# Railway-ге деплой (онлайн шығару)

Жоба **SQLite + Railway Volume** (тұрақты диск) арқылы деплой етіледі.
Дерекқорды Postgres-ке көшірудің қажеті жоқ — деректер деплой сайын сақталады.

## Алдын ала

- Railway аккаунт: https://railway.com

Екі әдіс бар. Бұл компьютерде **Git орнатылмаған**, сондықтан:

- **B әдіс (Railway CLI)** — Git қажет емес, ең жылдам. Төмендегі «B әдіс» бөлімін қараңыз.
- **A әдіс (GitHub)** — код жаңарған сайын автоматты деплой (ыңғайлырақ), бірақ алдымен
  Git орнату керек: https://git-scm.com/download/win

---

# B әдіс — Railway CLI (Git қажет емес) ⭐

1. Railway CLI орнату (терминалда):
   ```
   npm install -g @railway/cli
   ```
2. Кіру:
   ```
   railway login
   ```
3. Жоба папкасында жаңа Railway жобасын құру:
   ```
   railway init
   ```
4. Кодты жүктеп, деплой ету:
   ```
   railway up
   ```
5. Енді **3-қадам (Volume)** мен **4-қадам (Variables)** бөлімдерін Railway
   веб-панелінде орындаңыз, сосын қайта деплой:
   ```
   railway up
   ```
6. Домен ашу үшін **5-қадам**-ды қараңыз.

Кейін жаңарту: код өзгерткенде қайтадан `railway up` іске қосыңыз.

# A әдіс — GitHub арқылы (алдымен Git орнату керек)

## 1-қадам. Кодты GitHub-қа жүктеу

Терминалда жоба папкасында:

```
git init
git add .
git commit -m "Мектеп тапсырма платформасы"
```

GitHub-та жаңа (бос) repository ашып, көрсетілген нұсқаулықпен push жасаңыз:

```
git remote add origin https://github.com/ПАЙДАЛАНУШЫ/REPO.git
git branch -M main
git push -u origin main
```

## 2-қадам. Railway-де жоба құру

1. https://railway.com → **New Project** → **Deploy from GitHub repo** → осы repo-ны таңдаңыз.
2. Railway автоматты түрде Next.js-ті танып, build бастайды (алғашқы build сәтсіз болуы мүмкін — 4-қадамнан кейін қайта деплой етіледі).

## 3-қадам. Тұрақты диск (Volume) қосу

1. Сервисті ашып → **Settings** (немесе **Volumes**) → **Add Volume**.
2. **Mount path**: `/data`
   (Деректер осы дискіде сақталады — деплой сайын жоғалмайды.)

## 4-қадам. Айнымалыларды (Variables) орнату

Сервис → **Variables** → мыналарды қосыңыз:

| Айнымалы | Мәні |
|---|---|
| `DATABASE_URL` | `file:/data/prod.db` |
| `SESSION_SECRET` | ұзын кездейсоқ мән (мыс. 32+ таңба) |
| `ADMIN_USERNAME` | `admin` (қаласаңыз өзгертіңіз) |
| `ADMIN_PASSWORD` | **мықты құпия сөз** |
| `ADMIN_NAME` | `Жүйе әкімшісі` |

> `SESSION_SECRET` үшін кездейсоқ мән жасау (терминалда):
> `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`

## 5-қадам. Домен ашу

Сервис → **Settings** → **Networking** → **Generate Domain**.
Railway `https://...up.railway.app` сілтемесін береді — бұл сіздің онлайн мекенжайыңыз.

## 6-қадам. Деплой

Variables мен Volume орнатылғаннан кейін Railway автоматты қайта деплой етеді
(немесе **Deploy** батырмасын басыңыз).

Іске қосу кезінде автоматты орындалады (`npm run start:prod`):
1. `prisma migrate deploy` — дерекқор кестелерін құрады (алғашқы деплойда),
2. `node prisma/seed.mjs` — admin аккаунтын жасайды (бір рет),
3. `next start` — қосымшаны іске қосады.

## Дайын!

Сілтемеге кіріп, **admin** / (сіз қойған `ADMIN_PASSWORD`) арқылы кіріңіз.
Әрі қарай admin мектеп ашып, директор тағайындайды.

## Жаңарту (кейін код өзгертсеңіз)

```
git add .
git commit -m "өзгеріс сипаттамасы"
git push
```

Railway push-ты байқап, автоматты қайта деплой етеді. Деректер (Volume) сақталады.

## Ескертулер

- **Бір инстанс**: SQLite бір серверде жұмыс істейді (мектеп жүйесіне жеткілікті).
  Егер болашақта көп инстанс/ауқымдау керек болса — Postgres-ке көшу керек.
- **Сақтық көшірме**: маңызды деректер үшін Railway Volume-ды мезгіл-мезгіл
  жүктеп алып тұрған жөн (Railway CLI: `railway volume`).
